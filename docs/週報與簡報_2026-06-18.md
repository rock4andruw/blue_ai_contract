## 一、專案脈絡

### 起源

公司內部 AI 實作對決競賽，共 8 個題目，我們選定第 4 題：**合約文件比對與風險分析**。

競賽以 6、7、8 月大P月會為展示節點，評分標準：
- 技術性（1-10）
- 實用性（1-10）
- 商機推廣（1-10）

目標：**6 月月會 Demo，衝第一名**。

### 問題定義

每次供應商送來新版合約，法務或 PM 面臨三個問題：

1. **人工比對耗時**：逐條比對新舊版本，平均 92 分鐘
2. **風險判斷靠經驗**：哪些條款有風險、風險多高，需要法律知識
3. **協商對策要靠自己想**：談判桌上沒有現成建議

現有工具（如 Lumine AI）只列出差異，不告訴你哪個重要、風險多高、怎麼談。

### 我們的定位

> **「像有一個資深法務顧問幫你審合約」**

92 分鐘 → **30 分鐘以內**。不只找差異，給出重點摘要 + 風險等級 + 協商對策。

### Slide 6 — 我們的解法

**標題**：三層價值，一個流程

```
上傳兩份合約
     ↓
差異比對   → 找出所有新增／修改／刪除
     ↓
風險分析   → 高／中／低，附商業影響說明
     ↓
對策建議   → 每個高風險給 2-3 個協商方案
     ↓
完整報告   → 30 分鐘完成，可直接帶去談判
```

**核心原則**：Rule Engine 做判斷，LLM 做解釋。風險等級不交給 AI 猜。

---

### Slide 7 — 技術亮點

**標題**：可量化、可驗證、可解釋

**三個亮點**：

1. **Rule-based 風險引擎**
   - 11 條規則，涵蓋 SLA / 責任 / 保護條款 / 終止 / 保密等
   - High-risk recall：**100%**（38 筆 gold set 驗證）
   - 結果穩定可重現，不依賴 AI 黑箱

2. **LLM 只做解釋，不做判斷**
   - 接收結構化風險旗標 → 輸出白話摘要 + 協商對策
   - 降低幻覺風險，輸出可追溯

3. **無 API Key 也能跑**
   - Template fallback 機制，Demo 不受限
