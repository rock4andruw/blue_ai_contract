# 合約智能比對助理 — 專案藍圖摘要

**日期**: 2026-06-17  
**版本**: 2.1  
**定位**: 內部決策用，Demo 完整版（API + UI）已完成

---

## 一、專案定位

**題目**：合約文件（SLA／NDA／採購）差異比對與風險分析  
**核心價值**：不只找出差異，更告訴你為什麼重要、怎麼辦  
**對標競品**：Lumine AI（已有類似功能，但只列差異，缺乏重點整理與建議）  
**差異化**：濃縮成 3-5 個主要變更 + 風險等級 + 可直接用於協商的對策建議

## 三、技術架構（Phase 1 已實作）

```text
[上傳兩份合約 PDF/DOCX/MD]
        ↓
[Parser]          條款切分（clause_id + title + content）
        ↓
[Alignment]       條款對齊（LCS + 條款號 + Needleman-Wunsch DP）
        ↓
[Diff Engine]     新增 / 修改 / 刪除清單（DiffItem）
        ↓
[Risk Rule Engine] 規則判斷 → risk_flag / risk_level / trigger_reason
        ↓
[LLM Service]     白話摘要 / 重點收斂 / 協商對策（Claude API or template）
        ↓
[Report Generator] Markdown 報告輸出
        ↓
[FastAPI + Dashboard]  ← 待開發
```

**核心設計原則**：Risk Rule Engine 做判斷與標記，LLM 做解釋與表達。LLM 不決定風險等級，只把已標記的風險翻成白話並給出協商對策。

## 七、技術選型

| 元件 | 選擇 | 理由 |
| --- | --- | --- |
| LLM | Claude Sonnet 4.6 | 成本效益最佳，長文本理解強 |
| 文件解析 | pdfplumber + python-docx + MD parser | 支援三種格式 |
| 差異演算法 | difflib + Needleman-Wunsch DP | 純 Python，穩定可控 |
| 風險分類 | Rule-based（11 條規則） | 可測試、可解釋、可維護 |
| 向量 DB | PostgreSQL + pgvector | 不增加新系統（Phase 1.5） |
| 後端 | FastAPI（Python 3.11+） | 輕量、易整合 Claude API |
| 前端 | 靜態 HTML（Demo）→ React（正式） | Demo 先快速，正式再升級 |
| 認證 | Azure AD（現有） | 不導入 Keycloak |
| 部署 | Azure VM 單台 | 合約資料留在企業內網 |

**不採用知識圖譜**：現階段查詢不涉及跨合約關係遍歷，pgvector 足夠。
