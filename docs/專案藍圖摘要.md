# 合約智能比對助理 — 專案藍圖摘要

**日期**: 2026-06-26  
**版本**: 3.0  
**定位**: 內部決策用，Phase 1.5 完成（API + UI + MAS 雙重驗證 + 三層協商對策）

---

## 一、專案定位

**題目**：合約文件（SLA／NDA／採購）差異比對與風險分析  
**核心價值**：不只找出差異，更告訴你為什麼重要、怎麼辦  
**對標競品**：Lumine AI（已有類似功能，但只列差異，缺乏重點整理與建議）  
**差異化**：濃縮成 3-5 個主要變更 + 風險等級 + 可直接用於協商的對策建議

## 三、技術架構（Phase 1.5 已實作）

```text
[上傳兩份合約 PDF/DOCX/MD]
        ↓
[Parser]           條款切分（clause_id + title + content）
        ↓
[Alignment]        條款對齊（LCS + 條款號 + Needleman-Wunsch DP）
        ↓
[Diff Engine]      新增 / 修改 / 刪除清單（DiffItem）
        ↓
[Risk Rule Engine] 15 條規則 → risk_flag / risk_level / trigger_reason
        ↓
[LLM Service]      白話摘要 / 重點收斂 / 協商對策（Gemini 主 / Claude 備 / template）
        ↓
[MAS]              高風險 → Agent A（嚴格）‖ Agent B（平衡）→ Judge 矩陣
        ↓
[Report Generator] Markdown 報告 + MAS 雙重驗證標籤
        ↓
[FastAPI + Demo UI] ✅ 已完成
```

**核心設計原則**：Risk Rule Engine 做判斷與標記，LLM 做解釋與表達，MAS 做雙視角交叉驗證。LLM 不決定風險等級，只把已標記的風險翻成白話並給出協商對策。

## 七、技術選型

| 元件 | 選擇 | 理由 |
| --- | --- | --- |
| LLM | Gemini 3.1 Flash Lite（主）/ Claude Sonnet 4.6（備） | Gemini 免費額度高，Claude 備援 |
| 文件解析 | pdfplumber + python-docx + MD parser | 支援三種格式 |
| 差異演算法 | difflib + Needleman-Wunsch DP | 純 Python，穩定可控 |
| 風險分類 | Rule-based（15 條規則） | 可測試、可解釋、可維護 |
| MAS | ThreadPoolExecutor + Judge 矩陣 | 同一模型雙 Persona，Phase 2 改異質模型 |
| 後端 | FastAPI（Python 3.11+） | 輕量、Stateless |
| 前端 | 靜態 HTML（Demo）→ React（正式） | Demo 先快速，正式再升級 |
| 認證 | Azure AD（現有） | 不導入 Keycloak |
| 部署 | Azure VM 單台 | 合約資料留在企業內網 |
